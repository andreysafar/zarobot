# Zero-Bot Constructor - Product Requirements Document

## Project Overview
Zero-Bot Constructor - это платформа для создания персонализированных AI-ботов с NFT-владением и двухуровневой экономикой. Каждый бот является независимой единицей с собственным Telegram токеном и NFT паспортом.

## Core Concept
Конструктор ботов, где один код создает тысячи уникальных ботов из модулей личностей и инструментов. Каждый бот - это NFT актив, приносящий доход владельцу.

## Technology Stack
- Backend: Python 3.10+ с Django 4.2+
- Frontend/TMA: Langflow как веб-интерфейс управления
- Orchestration: Docker Swarm
- Database: MongoDB 6.0+ для конфигураций, Redis 7.0+ для кэша
- Blockchain: TON для NFT паспортов
- Monitoring: Grafana + Prometheus
- Hosting: Self-hosted → AWS (через iaMother в будущем)

## Key Features

### 1. Bot Factory System
- Создание независимых Telegram ботов с уникальными токенами
- Автоматический минтинг NFT паспортов на TON блокчейне
- Модульная система личностей и инструментов
- Docker Swarm для оркестрации контейнеров

### 2. Four Personality Modules (отдельные GitHub репозитории)
- **Iya (FREE)**: Базовая OpenAI личность с Langflow промптами
- **derValera (IAM-coin)**: Персональный помощник с Ollama дообучением
- **neJry (IAM-coin)**: Фитнес консультант с Computer Vision
- **neJny (IAM-coin)**: 18+ компаньон с ComfyUI генерацией

### 3. Dual Economy System
- **IAM-Coin**: B2B платежи от владельцев ботов к платформе
- **Stars**: B2C платежи от пользователей к владельцам ботов (100% доходов)

### 4. NFT Ownership System
- Каждый бот имеет NFT паспорт с метаданными (до 128KB)
- NFT определяет право владения и доступ к TMA консоли
- Торговля ботами через NFT transfer

### 5. TMA Management Console
- Langflow-based веб-интерфейс для владельцев ботов
- Настройка ценообразования Stars
- Аналитика доходов и пользователей
- Управление личностями и инструментами

## Critical Architecture Principles

### Data Isolation
- Данные изолированы по bot_id: /data/bots/{bot_id}/users/{user_id}/
- НЕ группировать по пользователям: /data/users/{user_id}/bots/
- Персональные модели derValera для каждой пары (bot_id, user_id)

### Telegram Integration
- Каждый бот = уникальный Telegram токен
- НЕ один общий бот для всех пользователей
- Webhook для каждого бота: /webhooks/telegram/{bot_id}

### Personality Architecture
- Личности = модули обработки сообщений
- НЕ отдельные боты или сервисы
- Общие сервисы с изоляцией данных по bot_id

### Economic Flow
- IAM: Владельцы ботов → Платформа (за создание, личности, инструменты)
- Stars: Пользователи → Владельцы ботов (за функции, 100% доходов)
- Платформа НЕ берет комиссию со Stars

## Development Phases

### Phase 1: MVP Infrastructure (1-2 months)
Core Django services, basic Iya personality, TMA console, Docker Swarm deployment

### Phase 2: Personality Modules (2-3 months)
derValera with Ollama, neJry port from existing MVP, neJny with ComfyUI

### Phase 3: Economy Integration (1-2 months)
Full IAM payments, Stars integration, NFT minting and verification

### Phase 4: Scaling (2-3 months)
AWS migration via iaMother, performance optimization, advanced analytics

## Technical Constraints
- Максимум 128KB для NFT метаданных
- Локальное хранение моделей и данных
- Docker Swarm (НЕ Kubernetes)
- MongoDB для конфигураций, Redis для кэша
- Grafana для мониторинга (НЕ внешние сервисы)
- Rate limiting: 100 запросов/минуту на пользователя

## Critical Implementation Rules
1. Данные изолированы по bot_id (НЕ по user_id)
2. Каждый бот имеет уникальный Telegram токен
3. Stars платежи идут владельцу бота (НЕ платформе)
4. IAM платежи идут платформе (НЕ владельцу бота)
5. Личности - модули, а НЕ отдельные боты
6. NFT содержит только метаданные (НЕ приватные данные)
7. Конфигурация токеномики в отдельном файле
8. Использовать Docker Swarm (НЕ Kubernetes)
9. Мониторинг через Grafana (НЕ внешние сервисы)
