# PRD: Zero-Bot MVP
## Минимально жизнеспособный продукт для платформы AI-ботов

**Версия:** MVP 1.0  
**Дата:** Январь 2025  
**Цель:** Создать базовую функциональную платформу для клонирования AI-ботов

---

## 1. ЦЕЛЬ MVP

Создать простую платформу, где пользователи могут:
1. Покупать личности (эхо-бот) и тулзы (веб-поиск) через IAMother бота
2. Создавать своих ботов с купленными компонентами
3. Использовать ботов за Stars

## 2. ОСНОВНЫЕ КОМПОНЕНТЫ MVP

### 2.1 IAMother Bot (Родительский бот)
- **Регистрация пользователей** в системе
- **Каталог личностей**: показ доступных личностей с ценами
- **Каталог тулзов**: показ доступных инструментов
- **Покупка за IAM**: обработка покупок личностей/тулзов
- **Управление кошельком**: пополнение и трата IAM токенов

### 2.2 Эхо-личность (Базовая личность)
- **UUID-паспорт**: уникальный идентификатор личности
- **Эхо-функция**: повторение сообщений пользователя
- **Ценообразование**: бесплатная базовая версия + платные функции

### 2.3 Веб-поиск тулз (Базовый инструмент)
- **UUID-паспорт**: уникальный идентификатор тулза
- **Поиск в интернете**: простой поиск через публичное API
- **Биллинг**: оплата за каждый поисковый запрос

### 2.4 Bot Factory (Система клонирования)
- **Создание ботов**: клонирование Docker контейнеров
- **Конфигурация**: настройка купленных личностей и тулзов
- **Деплой**: запуск готового бота для пользователя

### 2.5 Базовая экономика
- **Stars платежи**: конечные пользователи платят за функции
- **IAM токены**: владельцы ботов покупают улучшения
- **Простой биллинг**: учет трат без сложной аналитики

## 3. ПОЛЬЗОВАТЕЛЬСКИЕ ИСТОРИИ

### 3.1 Владелец бота
```
Как предприниматель
Я хочу создать AI-бота с эхо-личностью и веб-поиском
Чтобы предоставлять AI-сервисы пользователям за Stars
```

**Сценарий:**
1. Регистрируюсь через IAMother бота
2. Покупаю эхо-личность за IAM токены
3. Покупаю веб-поиск тулз за IAM токены
4. Создаю своего бота через Bot Factory
5. Получаю ready-to-use бота с уникальным токеном
6. Получаю доходы в Stars от пользователей

### 3.2 Конечный пользователь
```
Как обычный пользователь
Я хочу использовать AI-бота для общения и поиска
Чтобы получить быстрые ответы, платя Stars
```

**Сценарий:**
1. Нахожу бота через поиск в Telegram
2. Пишу сообщение -> получаю эхо-ответ
3. Запрашиваю поиск -> плачу Stars -> получаю результаты
4. Использую бота регулярно за небольшую плату

## 4. ТЕХНИЧЕСКАЯ АРХИТЕКТУРА MVP

### 4.1 Стек технологий
- **Backend**: Django + Django REST Framework
- **Database**: MongoDB (через MongoEngine)
- **Cache**: Redis
- **Telegram**: Telethon для взаимодействия с API
- **Контейнеризация**: Docker + docker-compose
- **Клонирование**: Docker API для создания новых экземпляров

### 4.2 Структура проекта
```
zero_bot/
├── apps/
│   ├── bot_factory/          # Создание и управление ботами
│   ├── message_router/       # Маршрутизация сообщений
│   ├── payments/             # Обработка Stars/IAM платежей
│   ├── tools_market/         # Каталог и продажа тулзов
│   └── iamother/             # NEW: IAMother бот
├── personalities/
│   ├── echo.py               # NEW: Эхо-личность
│   └── models.py             # NEW: Модели личностей
├── tools/
│   ├── web_search.py         # NEW: Веб-поиск
│   └── models.py             # NEW: Модели тулзов
├── integrations/

│       └── client.py         # NEW: Telethon клиент
└── core/
    └── models.py             # UPDATED: Упрощенные модели
```

## 5. КЛЮЧЕВЫЕ ТРЕБОВАНИЯ

### 5.1 Функциональные требования

**IAMother Bot:**
- [ ] Команда `/start` - регистрация пользователя
- [ ] Команда `/personalities` - каталог личностей
- [ ] Команда `/tools` - каталог тулзов  
- [ ] Команда `/buy_personality <uuid>` - покупка личности
- [ ] Команда `/buy_tool <uuid>` - покупка тулза
- [ ] Команда `/my_items` - список купленных предметов
- [ ] Команда `/create_bot` - создание нового бота
- [ ] Команда `/balance` - баланс IAM токенов

**Эхо-личность:**
- [ ] Обработка текстовых сообщений
- [ ] Возврат эхо-ответа с префиксом
- [ ] Учет использования для биллинга
- [ ] UUID-паспорт для идентификации

**Веб-поиск тулз:**
- [ ] Команда `/search <запрос>` в любом боте
- [ ] Поиск через бесплатное API (DuckDuckGo или подобное)
- [ ] Форматированный ответ с результатами
- [ ] Биллинг Stars за каждый запрос

**Bot Factory:**
- [ ] API для создания нового бота
- [ ] Клонирование базового Docker образа
- [ ] Настройка переменных окружения
- [ ] Запуск контейнера и получение токена
- [ ] Регистрация бота в базе данных

### 5.2 Нефункциональные требования

**Производительность:**
- Время создания нового бота: < 60 секунд
- Время ответа эхо-личности: < 1 секунда
- Время поиска: < 10 секунд

**Масштабируемость:**
- Поддержка до 100 одновременных ботов
- Возможность горизонтального масштабирования

**Безопасность:**
- Изоляция данных между ботами через bot_id
- Безопасное хранение токенов ботов
- Валидация всех пользовательских входов

## 6. ЭКОНОМИЧЕСКАЯ МОДЕЛЬ MVP

### 6.1 IAM Токены (для владельцев ботов)
- **Эхо-личность**: 100 IAM токенов (разовая покупка)
- **Веб-поиск тулз**: 50 IAM токенов (разовая покупка)
- **Создание бота**: 10 IAM токенов (за каждый новый бот)

### 6.2 Stars (для конечных пользователей)
- **Эхо-сообщение**: бесплатно (привлечение пользователей)
- **Веб-поиск**: 1 Star за запрос
- **Дневная подписка**: 10 Stars за безлимитный поиск

### 6.3 Распределение доходов
- **100% Stars** поступает владельцу бота
- **IAM платежи** поступают платформе

## 9. РИСКИ И ОГРАНИЧЕНИЯ

### 9.1 Технические риски
- Docker API может иметь ограничения на количество контейнеров
- MongoDB может потребовать оптимизации при росте числа ботов
- Telegram API имеет лимиты на количество сообщений

### 9.2 Бизнес-риски
- Простая эхо-личность может не привлечь пользователей
- Веб-поиск может быть недостаточно качественным
- Экономическая модель требует валидации рынком

### 9.3 Ограничения MVP
- Только одна личность и один тулз
- Отсутствие NFT и блокчейн интеграции
- Минимальная аналитика и мониторинг
- Простая экономическая модель
