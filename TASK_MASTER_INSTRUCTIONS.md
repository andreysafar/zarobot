## 🎨 СПЕЦИФИКА ПРОЕКТА ZERO-BOT

### Ключевые принципы:
1. **Изоляция данных**: каждый бот имеет уникальный `bot_id`
2. **UUID-паспорта**: личности и тулзы идентифицируются UUID
3. **Docker клонирование**: каждый бот = отдельный контейнер
4. **Двухуровневая экономика**: Stars (пользователи) + IAM (владельцы)

### Технические особенности:
- **MongoDB**: NoSQL для гибкого хранения данных ботов
- **Telethon**: асинхронный клиент для Telegram API
- **Docker API**: программное управление контейнерами
- **Redis**: кэширование и сессии

### Паттерны кодирования:
```python
# Все модели наследуются от BaseModel
class BotComponent(BaseModel):
    uuid_passport = fields.UUIDField(required=True, unique=True)
    bot_id = fields.StringField(required=True)  # Для изоляции данных

# Изоляция данных
def get_bot_data(bot_id: str, user_id: str):
    return f"/data/bots/{bot_id}/users/{user_id}/"
```
