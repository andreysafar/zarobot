# 🚀 Zero Bot Skills System - Быстрый запуск демонстрации

## 📋 Предварительные требования

- Python 3.12+
- pip или poetry
- SQLite (включен в Python)
- Telegram аккаунт для тестирования бота

## ⚡ Быстрый запуск (5 минут)

### 1. Установка зависимостей

```bash
# Установка Python зависимостей
pip install -r requirements.txt
```

### 2. Запуск демонстрации

```bash
# Автоматическая настройка и запуск
python run_demo.py
```

Этот скрипт автоматически:
- ✅ Применит миграции базы данных
- ✅ Создаст демонстрационные навыки и пользователей
- ✅ Соберет статические файлы
- ✅ Запустит Django сервер на http://localhost:8000

### 3. Тестирование IA-Mother бота (опционально)

В отдельном терминале:

```bash
# Запуск IA-Mother бота для Telegram
python test_ia_mother.py
```

Затем отправьте `/start` боту @IAMotherBot в Telegram.

## 🎪 Сценарий демонстрации

### 1. Django Admin (5 минут)
1. Откройте http://localhost:8000/admin/
2. Войдите как `admin` / `admin123`
3. Перейдите в **Skills** → **Skills**
4. Просмотрите созданные навыки
5. Откройте любой навык и покажите:
   - Метаданные и конфигурацию
   - Статистику установок и рейтинг
   - Систему модерации статусов

### 2. REST API (3 минуты)
1. Откройте http://localhost:8000/api/v1/skills/skills/
2. Покажите JSON список навыков
3. Откройте http://localhost:8000/api/v1/skills/categories/
4. Покажите категории с количеством навыков

### 3. IA-Mother Bot (7 минут)
1. Запустите `python test_ia_mother.py`
2. Откройте Telegram → @IAMotherBot
3. Отправьте `/start`
4. Покажите навигацию:
   - 🏪 Маркетплейс → 🧠 Навыки
   - 📂 По категориям
   - 🔥 Популярные навыки
   - 🔍 Детали навыка
   - 💾 Процесс установки

## 🐛 Устранение неполадок

### Проблема: Ошибка миграций
```bash
# Сброс миграций (ВНИМАНИЕ: удалит данные)
rm db.sqlite3
python manage.py migrate
python create_demo_skills.py
```

### Проблема: IA-Mother бот не отвечает
1. Проверьте токен в `.env`
2. Убедитесь, что бот запущен: `python test_ia_mother.py`
3. Проверьте интернет-соединение

### Проблема: Статические файлы не загружаются
```bash
python manage.py collectstatic --clear --noinput
```

### Проблема: Порт 8000 занят
```bash
# Запуск на другом порту
python manage.py runserver 0.0.0.0:8080
```
