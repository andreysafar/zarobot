## 1. Обзор продукта

### 1.1 Видение продукта
**Zero-Bot Constructor** - платформа для создания персонализированных AI-ботов, где каждый бот является NFT-активом с собственной экономикой, личностями и инструментами.

### 1.2 Постановка проблемы
- Существующие AI-боты не предоставляют владельцам возможность заработка
- Отсутствует реальное право собственности на AI-ассистентов
- Нет модульной системы для создания уникальных ботов
- Пользователи не могут монетизировать своих AI-помощников

### 1.3 Целевые пользователи

**Основные пользователи:**
- **Владельцы ботов (Предприниматели)** - создают, покупают и владеют различными личностями ботов и извлекают прибыль
- **Конечные пользователи** - используют функции ботов за Stars

**Вторичные пользователи:**
- **Разработчики инструментов** - создают инструменты для ботов
- **Администраторы платформы** - управляют экосистемой

## 2. Основные функции и требования

### 2.1 Система фабрики ботов

**ID требования:** BF-001  
**Приоритет:** P0 (Критический)  
**Описание:** Система создания ботов из модульных компонентов

**Критерии приемки:**
- [ ] Пользователь может приобрести бота (личность)
- [ ] При покупке бота автоматически создается его NFT паспорт
- [ ] Владелец получает уникальный bot_id для управления
- [ ] Поддержка различных тулзов приобретенных за iam владельцем
- [ ] раюотают подписки, как у владельца бота в iam, так и у конечного пользователя в stars

**История пользователя:**
```
Как владелец бота
Я хочу создать кастомного бота с определенной личностью и инструментами
Чтобы предлагать уникальные услуги пользователям и получать доход
```

### 2.3 Двухуровневая экономическая система

#### 2.3.1 Экономика IAM-Coin

**ID требования:** IAM-001  
**Приоритет:** P0 (Критический)  
P**Описание:** Оптовая экономика для владельцев ботов с подписками и разовыми платежами в IAM токенах

**Критерии приемки(экономика уточняется):**

**IAM Подписки (Ежемесячные лимиты):**
- [ ] **Базовый пакет** (1000 IAM/мес): 1 бот, 500 IAM на GPU, 300 IAM на LLM вызовы, 200 IAM на инструменты
- [ ] **Про пакет** (2500 IAM/мес): 3 бота, 1500 IAM на GPU, 700 IAM на LLM вызовы, 300 IAM на инструменты
- [ ] **Энтерпрайз пакет** (5000 IAM/мес): неограниченные боты, 3000 IAM на GPU, 1500 IAM на LLM вызовы, 500 IAM на инструменты

**IAM Разовые платежи:**
- [ ] Активация личности derValera: 50 IAM
- [ ] Активация личности neJry: 75 IAM  
- [ ] Активация личности neJny: 100 IAM
- [ ] Установка базового инструмента: 10-25 IAM
- [ ] Установка премиум инструмента: 50-100 IAM

**Потребление IAM токенов:**
- [ ] OpenAI GPT-4 вызов: 0.1 IAM за запрос
- [ ] Ollama обучение: 1 IAM за час GPU
- [ ] ComfyUI генерация: 0.5 IAM за изображение
- [ ] Computer Vision анализ: 0.2 IAM за изображение
- [ ] Веб-поиск: 0.05 IAM за запрос
- [ ] При превышении лимитов - дополнительная оплата по расценкам

**Техническая реализация:**
- [ ] Интеграция с TON кошельком для подписок и пополнений
- [ ] Автоматическое продление подписок
- [ ] Система учета трат IAM токенов в реальном времени
- [ ] Уведомления при достижении 80% лимита

## 3. Технические требования

### 3.1 Требования к производительности

**Время отклика:**
- Обработка сообщений: < 2 секунды (95-й процентиль)
- Верификация NFT: < 1 секунда
- Загрузка TMA панели: < 3 секунды

**Масштабируемость:**
- Поддержка 10,000+ одновременных пользователей на бота
- Автоматическое масштабирование 1-50 инстансов на личность
- Обработка 1,000+ ботов одновременно

### 3.2 Требования к безопасности

- Аутентификация на основе NFT для владения ботом
- Шифрование AES-256 для персональных данных
- HTTPS/WSS для всех коммуникаций
- Ограничение запросов: 100 запросов/минуту на пользователя

### 3.3 Требования к интеграции

**Блокчейн:**
- TON блокчейн для NFT смарт-контрактов
- TON Connect SDK для интеграции кошелька

**Telegram:**
- Telethon для обмена сообщениями бота
- Telegram Stars API для платежей
- Поддержка TMA (Telegram Mini App)

**AI/ML:**
- OpenAI API для личности Iya
- Ollama для тонкой настройки derValera
- ComfyUI для генерации контента neJny
- Computer Vision API для neJry

## 4. Требования к пользовательскому опыту

### 4.1 Процесс обнаружения бота

```
Пользователь находит бота → Начинает с бесплатного чата Iya → 
Узнает о платных личностях → Активирует через оплату Stars → 
Использует премиум функции → Возможно рекомендует друзьям
```

### 4.2 Путь владельца бота

```
Идея предпринимателя → Подключение TON кошелька → Создание конфигурации бота → 
Покупка личностей/инструментов за IAM → Настройка ценообразования → 
Запуск маркетинга → Получение дохода от Stars → Реинвестирование в улучшения
```

### 4.3 Процесс торговли ботами

```
Владелец решает продать → Размещает бота на маркетплейсе → 
Покупатель находит объявление → Проверка активов бота → 
Эскроу платеж → Передача NFT + активов → Обновление потока доходов


В данном контексте мы говорим про личности, любой пользователь может попытаться купить бота с которым общается в данные момент, оплата производится в iam коинах по фиксированному тарифу за сделку и в stars по тарифу опрделененному пользователем бота с комиссией 20% в сторону платформы.

все параметры токеномики и тарифы должны быть в отдельной конфигурации
```
